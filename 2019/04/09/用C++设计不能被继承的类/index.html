用C++设计一个不能被继承的类

发表于 2019-04-09 | 分类于 C++ | 阅读次数:
字数统计: 420

方法一. 定义静态函数

由于子类会调用父类的构造函数，一个类要想不能被继承，可以把它的构造函数和析构函数定义为私有函数，这样子类构造时就会报错。

但同时由于构造函数是私有的，也就无法创建实例了。这时需要再定义静态函数来创建和释放实例。

在这里，子类不可以继承A，同时可以通过A b = A::GetA() 来定义一个A的实例。（**因为成员函数GetA()是static的，所以可以通过类名直接调用。***）

方法二. 友元和模板

方法一只能局限于在堆上创建实例，并且通过这种额外定义函数的方式，感觉破坏了类的原有形式。更好的解决方式如下：

让A类再虚拟继承一个父类Afinal，这个父类的构造和析构函数是私有的，并且将A类声明为友元。那A类在创建实例的时候，就可以访问这个父类Afinal的构造和析构函数了，同时A类不允许继承，因为继承的子类会跳过A类，直接调用Ainal的构造函数，显然这样会发生错误，因为Afinal的构造函数是私有的。
